Qui sommes-nous

Architecture matérielle & logicielle

KubeSecureBox repose sur un nœud maître Raspberry Pi 4 (8 Go) et deux workers Pi 5 (8 Go), un disque 4 To en NFS, un switch Gigabit. Chaque Raspberry est configuré avec Ansible (sécurité, paramétrage réseau Tailscale, jointure au cluster Kubernetes). Kubernetes (K3s) et containerd pour l’orchestration, ArgoCD pour le GitOps, Prometheus/Grafana/Loki pour le monitoring, et les apps métier (pentesting, n8n) déployées dans le cluster.

Infra matérielle

Pi 4 (Control Plane + NFS 4 To), 2× Pi 5 (workers), switch Gigabit, cartes microSD SanDisk/Samsung, câbles Ethernet Cat 6A.

Sécurité & accès

Ubuntu Server 22.04 LTS, UFW, Fail2ban, Auditd, Lynis. Accès distant via VPN Tailscale (IP 100.x.x.x). Pas de ports exposés sur Internet.

FONCTIONNEMENT

GitOps, monitoring et pentesting

ÉTAPE 01

Manifestes Git → ArgoCD

Repo GitHub (YAML, Kustomize, Helm). ArgoCD synchronise et applique la config dans le cluster K3s (sync auto, self-heal, prune).

ÉTAPE 02

Métriques & logs

Prometheus, Node Exporter, kube-state-metrics. Loki + Promtail. Grafana pour dashboards et alertes Discord.

ÉTAPE 03

Scans & pentesting

App pentesting (Nmap, SQLMap, Hydra, etc.) exposée via Ingress. CronJobs pour scans planifiés.

FONCTIONNALITÉS

Stockage & réseau

NFS sur le Pi 4 pour le disque 4 To, NFS Provisioner pour PV/PVC. Tailscale pour l’accès distant, UFW et Fail2ban pour la sécurité.

  • NFS + StorageClass pour volumes persistants
  • Tailscale VPN, pas de ports exposés
  • UFW, Fail2ban, Auditd, Lynis
