Cluster Raspberry Pi
GitOps, monitoring & apps métier

KubeSecureBox est un cluster Kubernetes (K3s) sur Raspberry Pi. Chaque nœud est configuré avec Ansible (sécurité, réseau, kubernetes). GitOps (ArgoCD), monitoring (Prometheus, Grafana, Loki) et apps métier (pentesting, n8n). Accès distant via Tailscale.

À PROPOS

Infrastructure matérielle & logicielle

Infrastructure matérielle — Un nœud maître Raspberry Pi 4 (8 Go) et deux workers Pi 5 (8 Go), disque 4 To en NFS, switch Gigabit. Cartes microSD et câbles Ethernet pour l’interconnexion.

Infrastructure logicielle — Chaque nœud est provisionné avec Ansible (sécurité, Tailscale, jointure au cluster Kubernetes). Sur le cluster : Kubernetes (K3s), GitOps (ArgoCD), monitoring (Prometheus, Grafana, Loki). Apps métier : outil de pentesting, n8n (automatisation), parmi d'autres applications déployées sur le cluster.

BÉNÉFICES

Atouts & cas d’usage

Contrôle & reproductibilité

Tout le cluster est décrit en Git (YAML, Kustomize, Helm). Un push suffit pour déployer ou rollback : l’infra reste à portée de main, reproductible et versionnée.

Visibilité & réactivité

Dashboards et logs centralisés, alertes Discord dès qu’une anomalie est détectée. Surveillance en temps réel l’état du cluster sans ouvrir un terminal.

Lab sécurisé & self-hosted

Accès uniquement via Tailscale. Idéal pour apprendre Kubernetes, développer et déployer des applications métier en toute autonomie, sans dépendre du cloud.

FONCTIONNEMENT

GitOps, monitoring et apps

ÉTAPE 01

Manifestes Git → ArgoCD

Repo GitHub (YAML, Kustomize, Helm). ArgoCD synchronise et applique la config dans le cluster Kubernetes (K3s).

ÉTAPE 02

Métriques & logs

Prometheus scrape les métriques, Promtail envoie les logs vers Loki. Grafana pour visualisation et alerting Discord.

ÉTAPE 03

Apps métier

Déployées sur le cluster et exposées uniquement sur le réseau Tailscale. Chaque service est accessible en HTTPS via un Ingress, sur un sous-domaine de kubesecurebox.com.

FONCTIONNALITÉS

Stockage & réseau

NFS sur le Pi 4 pour le disque 4 To, NFS Provisioner pour créer des PV/PVC à la demande. Ansible déploie sur chaque Pi : Tailscale (accès distant 100.x.x.x), UFW, Fail2ban, et l’agent pour rejoindre le cluster Kubernetes.

  • NFS + StorageClass pour les volumes persistants
  • Tailscale VPN, pas de ports exposés sur Internet
  • UFW, Fail2ban, Auditd, Lynis

FONCTIONNALITÉS

Dashboards & alerting

Grafana s’appuie sur Prometheus et Loki comme sources de données. Dashboards dédiés : vue d’ensemble KubeSecureBox, métriques par nœud (CPU, RAM, disque, température), résumé des nœuds et exploration des logs.

Les alertes sont envoyées sur Discord (nœud indisponible, état NotReady, seuils RAM/CPU/disque, température, charge).

  • KubeSecureBox Overview, Node Summary, Loki Logs
  • Contact point Discord, règles kubesecurebox-infra
  • HTTPS via cert-manager
